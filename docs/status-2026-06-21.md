Notable engineering decisions baked into the above:
- Points are first-class shared entities ŌĆö no coordinate-matching/auto-merge. Entities connect only by deliberately sharing a Point id.
- Solving is explicit and batched (`POST .../solve`); nothing solves automatically on every edit.
- Constraint "blame" on non-convergence is a documented UX convention (most-recently-added), not a diagnosed root cause ŌĆö `py-slvs` itself flags every constraint in an inconsistent system.
- Circle's center/radius points deliberately don't join the Line-chain adjacency graph; mixed Line+Circle profile detection is an explicit, undone gap.
- `flutter_scene` was bumped `0.5.0-0` ŌåÆ `^0.18.1` this session (Flutter SDK now on the `master` channel) because the old version's Native Assets build hook is incompatible with master's current protocol. This drops the old `flutter_scene_importer` dependency chain entirely.

## What needs testing (real-device / human verification)

Every stage's PR description flags the same gap: this sandbox has no display, no Android emulator/iOS simulator, and no Windows host. Outstanding from real-device testing, oldest-first:

- **Stage 4**: real touch-drag feel (relative, scaled, Android/iOS) vs. real mouse (1:1, Windows); a live end-to-end run against the deployed backend with a real API key; actual Android/iOS/Windows builds.
- **Stage 5**: pinch-zoom / two-finger-pan gesture ergonomics on touch; scroll-wheel zoom / right-click-drag pan on Windows; on-screen rendering correctness (not just "builds a frame without throwing").
- **Stage 5 fixes**: does zoom-scaled cursor sensitivity actually *feel* right across the zoom range (formula is unit-tested, feel is not); visual clarity of the origin marker; Click button placement now that it's a separate persistent FAB.
- **Stage 6**: selection/hover highlighting and the contextual ribbon's visual layout/sizing on a real screen; delete UX (confirmation, error surfacing) under real touch.
- **Stage 7**: the 3D viewport's actual rendering output on a real GPU/device ŌĆö orbit/pan/zoom camera feel, mesh visual correctness, Feature-tree lock-state UI, and whether `flutter_scene` 0.18.1 performs acceptably on Android/iOS/Windows (only proven not-to-crash headlessly so far).

Known, explicitly out-of-scope gaps (not bugs, just undone): Arc entity, mixed Line+Circle profile detection, numeric/typed radius entry, zoom/pan persistence between sessions, dimension-editing UI, file save/load, an in-app settings UI for the API key.
