# DIDSA-CAD Status Summary — 2026-06-23

## What this covers

Stage 9: **Extrude (Boss + Cut)** — the first real OCCT geometry operation in
the tool, replacing the placeholder box mesh with actual solids. Both backend
and client are implemented and tested this session, on
`claude/didsa-cad-next-stage-dshvd7`.

## Current working state

End-to-end flow now works:

1. Draw a closed-loop Sketch (Lines/Circles) on a reference plane, as before.
2. Long-press the Sketch's row in the Feature tree. If its profile is a
   closed loop, an "Extrude" action is offered; otherwise it's shown
   disabled with the subtitle "Sketch does not contain a closed profile".
   The profile check (`GET /sketch/sketches/{id}/profile`) runs once per
   long-press, not on every render.
3. Tapping "Extrude" opens a bottom panel: Boss/Cut toggle (default Boss),
   Start distance (default 0.0) and End distance (default 10.0) fields,
   Confirm/Cancel buttons.
4. Editing either distance field debounces 500ms, then creates (first edit)
   or PATCHes (subsequent edits) an `ExtrudeFeature`, refetches the Part
   mesh, and renders it translucent/orange-tinted in the 3D viewport so the
   live preview is visually distinct from confirmed geometry.
5. **Confirm** commits the current values (even if the debounce never fired,
   e.g. distances were never touched after opening the panel) and refreshes
   the feature tree + mesh as normal, opaque geometry.
6. **Cancel** deletes the preview `ExtrudeFeature` and restores the mesh to
   its pre-extrude state.

## Backend (Stage 9)

- `ExtrudeFeature` model (`backend/app/document/models.py`): `extrude_type`
  (`boss`/`cut`), `start_distance`, `end_distance`, references the Sketch
  Feature it extrudes.
- `backend/app/document/extrude.py`: OCCT solid construction — builds the
  prism from the Sketch's closed wire via `BRepPrimAPI_MakePrism`, then
  fuses (`boss`) or cuts (`cut`) it against the accumulated solid from prior
  Extrude Features.
- New endpoints (`backend/app/document/router.py`): create/update/delete for
  `ExtrudeFeature`, under `/document/parts/{part_id}/extrude-features`.
- The Part mesh endpoint (`GET /document/parts/{id}/mesh`) is rewritten to
  tessellate the accumulated Extrude solid instead of returning the Stage
  8 placeholder box, falling back to the placeholder only when the Part has
  no Extrude Features yet.
- Tests: `backend/tests/test_stage9_extrude.py`, covering boss, cut, and the
  mesh endpoint's solid-vs-placeholder switch.
- **159/159 backend tests pass** (verified via the micromamba `cadtest` env —
  `pythonocc-core` is not importable in a bare `pip`/venv in this sandbox,
  so backend tests must run through micromamba; see prior session's status
  doc for the env setup, unchanged this session).

## Client (Stage 9)

- `client/lib/viewport3d/extrude_panel.dart` (new): the bottom-sheet form
  described above. Pure controlled widget — every keystroke reports
  `(type, start, end)` via `onChanged`; debouncing is `PartScreen`'s job.
- `client/lib/viewport3d/feature_context_menu.dart`: gained the `Extrude`
  list item (disabled + explanatory subtitle when the Sketch's profile
  isn't a closed loop).
- `client/lib/viewport3d/feature_tree_panel.dart`: `featureDisplayName` now
  takes the full Feature list (`Extrude 1`, `Sketch 2`, etc., counted per
  type) instead of just an index; tree rows show an extrude icon for
  Extrude Features.
- `client/lib/viewport3d/part_viewport.dart`: new `isPreviewMesh` flag —
  when true, the mesh's `UnlitMaterial` is swapped for a translucent
  orange-tinted one (`AlphaMode.blend`, `baseColorFactor` alpha 0.45).
- `client/lib/viewport3d/part_screen.dart`: owns all Extrude state
  (`_extrudeSketchFeature`, `_previewExtrudeFeatureId`, debounce timer,
  current type/distances) and the create-or-update-then-refetch-mesh flow;
  wires Confirm/Cancel.
- `client/lib/api/sketch_api_client.dart`: added `ProfileDetectionDto` and
  `SketchApiClient.getProfile`.
- Three new widget tests in `client/test/part_screen_test.dart`: closed-loop
  Extrude offered + confirmed, non-closed-loop Extrude disabled, and
  Cancel-after-live-preview deletes the preview Feature and restores the UI.
- **93/93 client tests pass**, `flutter analyze` clean.

## Known bugs fixed this session (client test infra, not app code)

- `_FakeDocumentBackend`'s test double always started its synthetic id
  counter at 1 regardless of seeded Features, so a newly-created
  `ExtrudeFeature` in a test seeding `feature-1` collided with that seed's
  id. Fixed by seeding the counter from `seedFeatures.length + 1`.
