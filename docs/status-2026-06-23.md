## Current working state

The full Stage 7/8 flow now works repeatedly, not just once:

1. Open `PartScreen` → tap a reference plane → "New Sketch on..." → 2D
   Sketch canvas opens, camera has animated to face that plane.
2. Draw Lines/Circles in the 2D canvas.
3. Leave the Sketch (back button, or the ribbon's now-wired "Exit Sketch").

   any Feature row reliably opens a sketch again — repeatable indefinitely,
   not just on the first attempt.

## Known bugs found this session — all fixed and device-confirmed

| Bug | Root cause | Fix |
|---|---|---|
| After exiting a Sketch, no further sketch could be created or opened — Feature appeared in the tree but the Sketch screen never opened | `PartViewportState` used `SingleTickerProviderStateMixin`, which only ever permits **one** `createTicker()` call for the State's entire lifetime — even after that ticker's `AnimationController` is disposed. `animateToPlane` builds a fresh `AnimationController(vsync: this, ...)` on every call (once per "enter a Sketch" action), so the *first* animation succeeds and every one after it throws on construction, silently rejecting the `Future` before `_openSketch` ever ran. Deterministic, not a race — explains "works once, then never again." | Switched `PartViewportState` to `TickerProviderStateMixin` (`part_viewport.dart`), which supports the repeated sequential `AnimationController`s this method actually needs. |
| Sketch geometry not consistently visible in the 3D viewport — seen once, not reliably | `_openSketch` never refreshed Feature/Sketch data after its `Navigator.push(...)` await resolved. The one time it *did* appear was a side effect of immediately creating another Feature afterward (which does call `_refreshSketchGeometries()`), not of returning from the Sketch itself. | `_openSketch` (`part_screen.dart`) now calls `_refreshFeatures()` + `_refreshSketchGeometries()` (via the existing `_runGuarded` wrapper) once the pushed route returns, regardless of which path triggered the pop. |
| "Exit Sketch" in the ribbon did nothing but show a placeholder snackbar — back button was the only real exit path | Never wired up; left as a TODO from Stage 6. | `sketch_ribbon.dart`'s "Exit Sketch" `ListTile` now calls `Navigator.of(context).pop()` — same pop path as the back button, so it benefits from the `_openSketch` refresh fix above for free. |

Fix commit: `0677c83` ("Fix sketch navigation breaking after the first
round-trip"), on `claude/plane-fixes-sketch3d-camera-anim`.
